Curve Finance是什么

Curve Finance类似Uniswap，也是一个去中心化的加密资产交易协议，不同的是在Uniswap上可以交易任意两种ERC-20代币，而Curve主要是用来交易Ethereum上的稳定币，例如USDT、USDC、DAI等。除了交易之外，Curve Finance还整合了其他协议，例如Balancer、Yearn.finance、Aave、Compound等，让流动性提供者提供的稳定币可以多重质押，增加收益。

Curve Finance的自动做市商（AMM）是如何设计的

在协议设计上，Curve Finance也是充分借鉴了前人的经验，例如Uniswap、Balancer等，如果大家对AMM是什么还不太清楚，可以参考文章Uniswap深度科普。

Curve Finance协议的定位是稳定币之间，或者是等价币之间（如ETH和sETH，BTC和wBTC等）的交易，这种类型的交易特点是代币的价格或者彼此之间的相对价格比较稳定，波动区间小，对应的AMM的设计就要有如下的特点：

  1. 在流动性池中的代币充足的情况下，滑点尽可能的小甚至没有；
  2. 避免流动性池中某一种代币被全部兑换，在某一种代币数量很少的时候要提升代币价格；

Curve Finance要做的就是找到一个恒等式，来实现一个满足上述两点要求的AMM。以流动性池中只有两种代币为例，如USDC/DAI，交易价格是由-dy/dx来决定的，如果让交易价格稳定为1，最简单直接的恒等式就是采用一个斜率为-1的直线，如：

x+y=constx + y = const

但在实际的交易市场中，即使是稳定币价格也会出现波动，所以还是需要考虑到一个价格自调整的能力，像Uniswap、Bancor等协议，是通过乘积恒等式来实现价格调整，如：

x∗y=constx * y = const

在这种模型下，价格滑点会比较明显，单笔交易量越大，价格变化越大，因此不太适合稳定币之间的交易。

综上分析，Curve Finance是需要找到两种AMM类型之间的一个恒等式，既能创造比较好的流动性，有一定的价格自调整能力，又能在给定的交易范围内降低滑点，达到的效果如下图所示（取自StableSwap白皮书）：

上图中的蓝色曲线即代表稳定币之间交易的AMM曲线，和Uniswap这种乘积恒等式的AMM设计来比，在预期的稳定价格附近有比较小的滑点，和恒定总和的AMM设计相比，在某一种代币余额产生较大偏离时，交易价格会发生比较大的变化，促使流动性提供者或者套利者通过市场行为恢复流动性池中代币余额的均衡。

稳定币交易恒等式的构建

先考虑只有两个币种的场景，假设恒定总和与恒定乘积的恒等式如下：

x+y=Dx + y = D

x∗y=k x * y = k

在比较理想的初始状态下，X和Y两个币种的余额是相等的，即x = y = D/2，则乘积恒等式可表示为：

x∗y=(D/2)2x * y = (D/2)^2

为了获得这两种AMM的特点，Curve把两种恒等式做了一个整合形成一个联合曲线，为了更灵活有效的调节恒等式的属性，给恒定总和恒等式加了一个因子 χ\chi ，等到如下恒等式：

χ(x+y)+xy=χD+(D/2)2\chi(x + y ) + xy = \chi D + (D/2)^2

χ\chi 的作用可看作是用来调整恒定总和恒等式影响占比的因子，值越大，则曲线更加倾向于零滑点的直线，值越小，则曲线更加倾向于高滑点的恒定乘积曲线，我们可以看一个曲线变化的动态图：

从上图可以看出随着 χ\chi 的增大，组合曲线越来越接近于直线，滑点越小，这样我们就可以通过调节 χ\chi 来控制组合曲线的表现，也就是我们需要一个动态因子 χ\chi ，在curve finance的白皮书中也成为杠杆（leverage），根据上面的分析，这个动态杠杆需要满足两个特性：

  1. 自适应梯度，当流动性池中的各个代币余额相等或者接近的时候， χ\chi 的值增大，降低滑点，反之值减小，增加曲线梯度，即交易价格，控制余额比例的偏离；
  2. 与交易池中代币种类不相关，即与维度不相关。

上文提到的AMM的例子中，流动性池中只有两种代币，即维度是2，为了使 χ\chi 与维度无关，可以在恒定和的等式两边均乘以D，使得恒定和与恒定积有相同的维度，即 Dχ(x+y)=D2D\chi(x +y)=D^2 ，则联合曲线的恒等式可改写为：

Dχ(x+y)+xy=χD2+D2/22D \chi(x + y) + xy = \chi D^2 + D^2 / 2^2

接下来还要满足 χ\chi 要能根据流动性池中代币之间的余额比例来自适应的变化，即各个代币余额越接近 χ\chi 越大，反之越小，则可根据均值不等式的原理，即和一定时，各个因子相等时积最大，来定义一个 χ\chi 的公式，如下：

χ=Axy(D/2)2 \chi = \frac{Axy}{(D/2)^2}

其中A为一个常数，即当x=y=D/2时， χ\chi 是一个常数，同时也是最大值，当代币之间的余额比例偏离平衡时， χ\chi 变小，而且偏离的越远 χ\chi 的值越小，这样就满足了动态杠杆需要的两个特性。

最终获得的流动性池中只有两种代币的恒等式如下：

DAxy(D/2)2(x+y)+xy=Axy(D/2)2D2+D222D\frac{Axy}{(D/2)^2}(x+y) + xy = \frac{Axy}{(D/2)^2}D^2 + \frac{D^2}{2^2}

即

22A(x+y)+D=22AD+D322xy 2^2A(x+y) + D = 2^2AD + \frac{D^3}{2^2xy}

以上举得例子是2维空间的，也就是流动性池中只有两种代币，把以上共识扩展到n维空间的话，就可以得到Curve Finance最终使用的联合曲线的恒等式:

Ann∑xi+D=ADnn+Dn+1nn∏xiAn^n\sum x_i + D = ADn^n + \frac{D^{n+1}}{n^n\prod x_i}

在这个恒等式中，A的值是在创建流动性池的时候选择的，恒定不变，D的值在质押代币之后需进行计算，并且在swap过程中要不断的进行迭代更新。

流动性池的种类

Curve Finance与Uniswap这种DeFi交易协议还有一个很大的不同，就是它原生就支持与其他DeFi协议的融合，Curve的流动性池也因此分为三种类型：

  1. Plain pools：两个或多个稳定币交易流动性池，纯粹的用于交易的流动性池；
  2. Lending pools：在这种交易池中，对原生的代币进行了一个再包装（wrapped tokens），像是一个代币的代币，例如用户质押的是DAI，但实际流动性池里是cDAI，这样做的目的是为了可以将质押到流动性池中的代币再借贷出去赚取利润，而用于交易的流动性池中实际保存的是wrapped tokens；
  3. Metapools：稳定币与另外一个流动性池中的流动性代币（LP）组成的流动性池，这种流动性池的目的是为了给用户一个重叠质押的机会，例如用户质押DAI到一个Plain pool获得LP，然后又将LP质押到Metapool继续赚取收益，就像是你买了债券，然后又用债券换了一个理财，这样你既能获得债券的收益，又能获得理财的收益。

总结

正因为Curve Finance提供了类似加杠杆的重复质押的功能，后续又衍生出了机枪池这种为用户寻求最佳收益的产品或者协议，关于机枪池以及Curve Finance不同流动性池在实际应用中的使用，每一个方面都可以深入探讨，限于篇幅就不在这里做过多介绍了，作为一个稳定币的DeFi协议，Curve Finance为上层的应用或者衍生品提供了基础支撑，相信未来肯定还会有更多的创新性的尝试，包括跟元宇宙的结合等等，我们拭目以待吧。
